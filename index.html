Prompt Engineering for Coding

Level: Grade 6+

1. Introduction

Artificial Intelligence (AI) tools like coding assistants can generate programs, fix errors, and explain code. However, the quality of the result depends entirely on how we ask the question.

If you ask a vague question, you get a vague answer. If you ask a clear, detailed question, you get a much better result. This skill of writing clear and effective instructions for AI is called:

💡 PROMPT ENGINEERING

In this lesson you will learn:

  • What a prompt is
  • What prompt engineering means
  • How to write weak vs strong prompts
  • The Prompt Power Formula
  • How to use AI coding tools responsibly
Prompt Engineering

2. What is a Prompt?

A prompt is the instruction or message you give to an AI system. It is how you communicate with AI — just like how you would give instructions to another person.

Example of a Simple (Weak) Prompt:
Create a website.
⚠ This is a weak prompt because it is not clear. The AI does not know: What kind of website? What content? What language? What design?
TypeExampleResult
✗ Weak Prompt Create a website. AI may produce anything — usually not what you wanted.
✓ Strong Prompt Create a simple school homepage using HTML and CSS with a nav bar, welcome message, and blue color scheme. AI produces exactly what was described.
What is a Prompt?

3. What is Prompt Engineering?

Prompt Engineering means writing clear, detailed, and structured instructions to get better results from AI.

💡 Remember: AI does not read minds. It only understands what we type. The more specific you are, the better the result.

Think of it like giving directions to someone:

  • 🚫 Bad direction: "Go somewhere near the school."
  • ✅ Good direction: "Turn left at the main gate, walk 50 metres, enter the second building on the right."

The same principle applies to writing prompts for AI!

Prompt Engineering

4. Why Prompt Engineering is Important in Coding

When using AI for coding, the quality of the prompt directly affects the quality of the code:

Prompt QualityCode Result
✗ Weak prompt Weak, incomplete, confusing code
✓ Clear prompt Better, organised, working code
Good Prompt Engineering Helps You:
  • ✅ Save time — get the right code on the first try
  • ✅ Reduce mistakes — AI understands exactly what you need
  • ✅ Generate cleaner code — well-structured, readable output
  • ✅ Understand the solution better — ask AI to explain its code

5. Weak Prompt vs Strong Prompt

✗ WEAK PROMPT
Make calculator
Problems:
  • Which programming language?
  • What type of calculator?
  • Who is it for?
  • How complex should it be?
  • No explanation requested
  • No design details given
✓ STRONG PROMPT
Create a simple calculator using HTML and JavaScript. Requirements: - Two input boxes - Add and subtract buttons - Display result below - Simple design for beginners - Add comments in the code - Explain how the code works
Why it is better:
  • Specifies the language (HTML + JavaScript)
  • Lists exact features needed
  • Sets difficulty level (beginner)
  • Asks for code comments
  • Requests an explanation
A strong prompt is: ✔ Clear   ✔ Detailed   ✔ Structured

6. The Prompt Power Formula

A strong coding prompt usually includes 5 key parts. Follow this formula for best results:

1
🌟 Role — Tell AI who it should act as
Giving AI a role helps it produce more appropriate, focused answers.
Example: "You are a coding teacher helping a Grade 6 student."
2
🎯 Task — Clearly describe what you want
Be specific about the type of program or answer you need.
Example: "Create a number guessing game in Python."
3
📋 Requirements — List what must be included
Bullet points work well here. Be as specific as possible.
Example: - Use random numbers between 1 and 10 - Give the player 3 chances - Print whether they won or lost
4
🚫 Constraints — Set limits or conditions
Tell AI what NOT to include or what boundaries to stay within.
Example: - Keep it beginner friendly - Do not use advanced functions or libraries
5
📄 Output Format — Ask for explanation or formatting
Tell AI how you want the answer structured.
Example: "Explain the code step by step after the program."
Prompt Power Formula

Full Example – Using the Prompt Power Formula

Here is a complete strong prompt built using all 5 parts of the formula:

You are a coding teacher helping a Grade 6 beginner student. [ROLE]

Create a number guessing game in Python. [TASK]

Requirements: [REQUIREMENTS]
- Use random numbers between 1 and 10
- Give the player 3 chances to guess
- Tell the player if their guess is too high or too low
- Print a congratulations message if they win
- Print the correct answer if they run out of chances

Keep it beginner friendly and do not use classes
or advanced functions. [CONSTRAINTS]

After the code, explain each section step by step
in simple English. [OUTPUT FORMAT]
💡 Result: This prompt will produce clean, well-commented Python code with a full step-by-step explanation — perfect for a beginner student!

7. How AI Coding Assistants Work

✅ AI Coding Assistants DO:
  • Learn from large amounts of code
  • Recognize patterns in programming
  • Predict what should come next
  • Generate code based on your instructions
✗ AI Coding Assistants DO NOT:
  • Think like humans
  • Understand emotions or context
  • Always give correct answers
  • Know your full project without being told
👉 AI coding assistants are prediction tools, not human programmers. They predict the most likely correct answer — but they can and do make mistakes!
How AI Works

8. Common Mistakes in Prompt Writing

Students often make these common mistakes when writing prompts for AI coding tools:

✗
Writing very short prompts — "Make login page" gives AI almost no information to work with.
✗
Forgetting to mention the programming language — AI may choose Python when you needed JavaScript!
✗
Not asking for an explanation — You receive code but do not understand how it works.
✗
Not mentioning skill level — AI may produce advanced code that is impossible for a beginner to understand.
✗
Forgetting to specify design or output — AI does not know what the result should look like unless you tell it.
✓ Quick Fix: Before sending a prompt, ask yourself — "Does AI have enough information to give me exactly what I want?" If not — add more details!
Common Mistakes

9. Responsible Use of AI in Coding

AI is a tool — just like a calculator or a dictionary. Using it well means using it to support your learning, not replace it.

✅
Read and understand every line of code AI gives you before using it.
✅
Test the program yourself — run it, check if it works as expected.
✅
Modify the code yourself — change variable names, add features, make it your own.
✅
Never copy without learning — understanding matters more than the output.
💡 Rule: AI should assist your brain — not replace it.
